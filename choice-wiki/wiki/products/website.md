## TL;DR
Modern restaurant website with own domain, menu editor, automatic translation, SEO, and connections to QR / delivery / reservations. **Available from BASIC plan.** Main pain: no website or old one, menu managed in multiple places. Main argument: one editor for web, delivery, QR, and reservations.

## What It Is
A simple, visually attractive restaurant website with own domain, automatic translation, menu editor, SEO, and integrations (Google Analytics, Facebook Pixel).

## Who It's For
- No website, or old one that displays poorly on mobile
- Managing menu separately on website, delivery platforms, and QR
- Paying an IT firm for every small change / fragmented suppliers

## Key Features
- **Mobile-first design** — 90% of users browse via mobile; fully responsive
- **Integrated ordering** — Delivery and takeaway built in, no third-party plugins
- **Own domain** — e.g. myrestaurant.com
- **SEO optimized** — built-in tools, meta tags, Google integration
- **99-language auto-translation** — international guests served instantly
- **Promotional banners** — deals, happy hours, seasonal offers
- **POS integration** — orders flow directly to Poster, Syrve, R-Keeper, Storyous, Dotykačka, etc.
- **Payment methods** — bank cards, Apple Pay, Google Pay
- **Multi-location** — one admin panel for all branches
- **One editor for everything** — changes to menu sync to website, QR, delivery, and platforms
